#### 2.1.2 索引设计

| 索引名 | 索引类型 | 字段 | 说明 |
|--------|----------|------|------|
| pk_projects | 主键索引 | id | 主键自增索引 |
| uq_certificate_number | 唯一索引 | certificate_number | 参选证号码唯一约束 |

#### 2.1.3 建表SQL

```sql
CREATE TABLE projects (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_name VARCHAR(500) NOT NULL,
    project_cost NUMERIC(12, 4) DEFAULT 0,
    declaring_company VARCHAR(500),
    project_manager VARCHAR(100),
    contact_phone VARCHAR(200),
    supervision_company VARCHAR(500),
    project_director VARCHAR(100),
    certificate_number VARCHAR(50) NOT NULL UNIQUE,
    participating_company VARCHAR(500),
    creation_target VARCHAR(100),
    region VARCHAR(50),
    construction_unit VARCHAR(500),
    start_date DATE,
    end_date DATE,
    standardization_application_date DATE,
    construction_permit_date DATE,
    city_standardization_inspection_date DATE,
    inspectors VARCHAR(200),
    remarks TEXT,
    status VARCHAR(50) NOT NULL DEFAULT '正常',
    pre_score NUMERIC(5, 2) DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE UNIQUE INDEX uq_certificate_number ON projects(certificate_number);
```

## 6. 数据完整性约束

### 6.1 实体完整性
- 主键约束：id字段自增，唯一标识每条记录
- 非空约束：project_name、certificate_number、status字段不允许为空

### 6.2 参照完整性
- 当前系统为单表设计，无外键关联

### 6.3 域完整性
- 数据类型约束
- 默认值约束
- 唯一性约束（certificate_number）
- 检查约束（通过应用层实现）

## 7. 数据备份策略

### 7.1 备份方式
- 全量备份：复制SQLite数据库文件
- 备份文件命名：`backup_YYYYMMDDHHMMSS.db`

### 7.2 备份存储
- 存储目录：`data/backups/`
- 自动备份：支持按日/周/月周期自动备份
- 保留策略：可配置保留备份数量

### 7.3 恢复方式
- 上传备份文件覆盖当前数据库
- 支持通过系统设置页面操作
- 恢复操作需要密码验证

## 8. 性能优化建议

### 8.1 索引优化
- 已对certificate_number建立唯一索引
- 可根据查询频率考虑对以下字段建立索引：
  - region（区域筛选）
  - status（状态筛选）
  - construction_permit_date（排序查询）

### 8.2 查询优化
- 使用分页查询避免全表扫描
- 合理使用WHERE条件过滤
- 避免SELECT *，按需查询字段

### 8.3 数据量预估
- 单表记录数：预计10,000条以内
- 单条记录大小：约1KB
- 数据库文件大小：预计10MB以内

## 10. 附录

### 10.1 SQLAlchemy模型定义

```python
class Project(db.Model):
    __tablename__ = 'projects'
    
    id = db.Column(db.Integer, primary_key=True)
    project_name = db.Column(db.String(500), nullable=False)
    project_cost = db.Column(db.Numeric(12, 4), default=0)
    declaring_company = db.Column(db.String(500))
    project_manager = db.Column(db.String(100))
    contact_phone = db.Column(db.String(200))
    supervision_company = db.Column(db.String(500))
    project_director = db.Column(db.String(100))
    certificate_number = db.Column(db.String(50), unique=True, nullable=False)
    participating_company = db.Column(db.String(500))
    creation_target = db.Column(db.String(100))
    region = db.Column(db.String(50))
    construction_unit = db.Column(db.String(500))
    start_date = db.Column(db.Date)
    end_date = db.Column(db.Date)
    standardization_application_date = db.Column(db.Date)
    construction_permit_date = db.Column(db.Date)
    city_standardization_inspection_date = db.Column(db.Date)
    inspectors = db.Column(db.String(200))
    remarks = db.Column(db.Text)
    status = db.Column(db.String(50), nullable=False, default='正常')
    pre_score = db.Column(db.Numeric(5, 2), default=0)
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    updated_at = db.Column(db.DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
    
    def __repr__(self):
        return f'<Project {self.project_name}>'
```
