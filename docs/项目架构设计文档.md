# 工程项目管理系统 - 项目架构设计文档

## 1. 项目概述

### 1.1 项目名称
工程项目管理系统（Project Management System）

### 1.2 项目目标
开发一个无需登录验证的工程项目管理系统，实现对工程项目信息的全生命周期管理，包括项目录入、查询、统计、导入导出等功能。系统集成OCR智能识别功能，支持从图片自动提取项目信息。

### 1.3 技术选型

| 层次 | 技术栈 | 版本要求 |
|------|--------|----------|
| 后端框架 | Flask | 2.3.3 |
| 数据库 | SQLite | 3.x |
| ORM框架 | Flask-SQLAlchemy | 3.0.5 |
| OCR引擎 | RapidOCR | latest |
| 前端框架 | Bootstrap | 5.3.0 |
| 图表库 | Chart.js | 2.9.4 |
| 图标库 | Feather Icons | 4.29.0 |
| Python版本 | Python | 3.7.3+ |

### 1.4 运行环境要求
- 操作系统：UOS / Windows / Linux / macOS
- CPU架构：ARM / x86_64
- Python版本：3.7.3及以上

### 2.2 架构说明

系统采用经典的三层架构设计：

1. **表现层（Presentation Layer）**
   - 基于Jinja2模板引擎渲染HTML页面
   - 使用Bootstrap 5实现响应式布局
   - Chart.js实现数据可视化
   - 支持OCR图片上传交互
   - 自定义Toast消息提示组件
   - 密码验证弹窗组件

2. **业务逻辑层（Business Logic Layer）**
   - Flask路由处理HTTP请求
   - 实现业务规则和数据验证
   - 提供RESTful API接口
   - 集成OCR识别服务
   - 敏感操作密码验证机制
   - 自动备份调度器

3. **数据访问层（Data Access Layer）**
   - SQLAlchemy ORM实现对象关系映射
   - 封装数据库操作
   - 保证数据一致性

```

│                     OCR识别处理流程                          │

│                                                             │
│  1. 用户上传图片                                             │
│           ↓                                                 │
│  2. 前端发送图片到 /api/ocr                                  │
│           ↓                                                 │
│  3. 后端接收并验证图片格式                                   │

│                                                             │
```

## 8. 安全设计

### 8.1 安全措施

| 安全项 | 实现方式 |
|--------|----------|
| 敏感操作验证 | 管理密码验证机制 |
| XSS防护 | Jinja2自动转义 |
| SQL注入防护 | SQLAlchemy参数化查询 |
| 密钥管理 | 配置文件SECRET_KEY |
| 文件上传 | 类型验证、大小限制 |
| OCR上传 | 仅允许图片格式，限制文件大小 |
| 密码存储 | bcrypt哈希加密 |

### 8.2 敏感操作保护

以下操作需要验证管理密码：
- 删除项目
- 数据库恢复
- 清除敏感认证状态

密码验证有效期：1分钟

### 8.3 数据验证

- 前端验证：HTML5表单验证 + JavaScript验证
- 后端验证：Python数据类型检查 + 业务规则验证
- 数据库约束：NOT NULL、UNIQUE约束

## 9. 性能优化

### 9.1 前端优化
- 静态资源压缩（CSS/JS）
- 图标使用SVG格式
- 图表按需加载
- OCR上传异步处理

### 9.2 后端优化
- 数据库查询优化（索引、分页）
- ORM懒加载
- 上下文处理器减少重复计算
- OCR实例缓存（单例模式）
- 图片自动缩放减少处理时间

### 9.3 数据库优化
- 合理设置字段类型和长度
- 唯一索引（参选证号码）
- 分页查询避免全表扫描

### 9.4 OCR性能优化
- 图片缩放至1280x1280以内
- OCR实例缓存避免重复初始化
- 平均识别时间约1.7秒
