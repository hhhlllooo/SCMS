## 3. 安装部署

### 3.1 获取项目

```bash
# 克隆项目（如果使用Git）
git clone <repository_url>
cd projext3
```

### 3.2 安装依赖

```bash
# 安装Python依赖
pip install -r requirements.txt
```

### 3.3 依赖说明

| 依赖包 | 版本 | 用途 |
|--------|------|------|
| Flask | 2.3.3 | Web框架 |
| Flask-SQLAlchemy | 3.0.5 | ORM框架 |
| Werkzeug | 2.3.7 | WSGI工具库 |
| rapidocr-onnxruntime | latest | OCR识别引擎 |
| Pillow | latest | 图像处理 |
| numpy | latest | 数值计算 |
| bcrypt | latest | 密码加密 |

### 3.4 初始化数据库

```bash
# 数据库会在首次运行时自动创建
# 或手动初始化
python -c "from app import app, db; app.app_context().push(); db.create_all()"
```

系统配置位于 `app.py` 文件中：

```python
# 基础配置

app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///projects.db'
app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = False
app.config['SECRET_KEY'] = 'your-secret-key-here'

# 上传配置
app.config['MAX_CONTENT_LENGTH'] = 16 * 1024 * 1024  # 16MB
app.config['UPLOAD_FOLDER'] = 'uploads'
```

## 4. 运行应用

## 6. 数据备份

### 6.1 手动备份

1. 通过系统设置页面备份
   - 访问"系统设置"页面
   - 点击"立即备份"按钮

2. 命令行备份
   ```bash
   # 复制数据库文件
   cp data/scms.db data/backups/backup_$(date +%Y%m%d%H%M%S).db
   ```

### 6.2 自动备份

系统支持定时自动备份功能：

1. 访问"系统设置"页面
2. 开启"自动备份"开关
3. 选择备份周期（日/周/月）
4. 设置备份时间
5. 保存配置

### 6.3 数据恢复

1. 通过系统设置页面恢复
   - 访问"系统设置"页面
   - 选择备份文件或上传新文件
   - 输入管理密码验证
   - 确认恢复

2. 命令行恢复
   ```bash
   # 停止应用
   # 复制备份文件
   cp data/backups/backup_xxx.db data/scms.db
   # 重启应用
   ```

## 7. 安全配置

### 7.1 密钥配置

通过环境变量或 `app/config/settings.py` 配置密钥：

```bash
# 推荐：使用环境变量
export SECRET_KEY=$(python -c "import secrets; print(secrets.token_hex(32))")
```

或在 `.env` 文件中配置：
```
SECRET_KEY=your-production-secret-key
```

### 7.2 管理密码

管理密码哈希通过环境变量配置：

```bash
# 生成密码哈希
python -c "import bcrypt; print(bcrypt.hashpw('your_password'.encode('utf-8'), bcrypt.gensalt()).decode('utf-8'))"

# 设置环境变量
export SENSITIVE_PASSWORD_HASH='$2b$12$...'
```

或在 `.env` 文件中配置：
```
SENSITIVE_PASSWORD_HASH=$2b$12$...
```

### 7.3 文件上传安全

```python
# 限制上传文件大小
app.config['MAX_CONTENT_LENGTH'] = 16 * 1024 * 1024  # 16MB

# 允许的文件类型
ALLOWED_EXTENSIONS = {'png', 'jpg', 'jpeg', 'csv', 'db'}
```

## 10. 常见问题

### 10.1 端口被占用

**问题**：启动时提示端口5000被占用

**解决方案**：
```bash
# Windows：查找并结束占用端口的进程
netstat -ano | findstr :5000
taskkill /F /PID <PID>

# Linux/macOS
lsof -i :5000
kill -9 <PID>

# 或使用其他端口
python app.py --port 5001
```

### 10.2 模块导入错误

**问题**：启动时提示模块未找到

**解决方案**：
```bash
# 确保已安装所有依赖
pip install -r requirements.txt

# 或单独安装缺失模块
pip install <module_name>
```

### 10.3 数据库锁定

**问题**：操作时提示数据库锁定

**解决方案**：
- 确保没有其他进程访问数据库
- 检查数据库文件权限
- 重启应用

### 10.4 OCR识别失败

**问题**：OCR识别功能无法使用

**解决方案**：
```bash
# 确保安装了OCR依赖
pip install rapidocr-onnxruntime

# 检查图片格式和大小
# 支持格式：jpg、jpeg、png
# 大小限制：16MB
```

### 10.5 编码问题

**问题**：CSV导入/导出乱码

**解决方案**：
- 确保CSV文件使用UTF-8编码
- Windows下可使用UTF-8 with BOM

## 12. 生产环境部署建议

### 12.1 使用反向代理

推荐使用Nginx作为反向代理：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /static {
        alias /path/to/projext3/static;
        expires 30d;
    }
}
```

### 12.2 使用Systemd服务（Linux）

创建服务文件 `/etc/systemd/system/project-management.service`：

```ini
[Unit]
Description=Project Management System
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/path/to/projext3
ExecStart=/path/to/venv/bin/gunicorn -w 4 -b 127.0.0.1:5000 app:app
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl start project-management
sudo systemctl enable project-management
```

### 12.3 使用Docker部署

创建 `Dockerfile`：

```dockerfile
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 5000

CMD ["gunicorn", "-w", "4", "-b", "0.0.0.0:5000", "app:app"]
```

构建并运行：

```bash
docker build -t project-management .
docker run -d -p 5000:5000 -v $(pwd)/instance:/app/instance project-management
```

## 9. 模块化架构说明

### 9.1 重构背景

原项目采用单文件架构（`app.py`），随着功能增加，代码量达到 1500+ 行，存在以下问题：
- 职责不分离，路由、模型、业务逻辑混在一起
- 难以维护和扩展
- 测试困难
- 不利于团队协作

### 9.2 重构目标

- **模块化**：按功能拆分为独立的模块
- **职责分离**：路由、业务逻辑、数据访问分层
- **可维护性**：代码结构清晰，易于理解和修改
- **可扩展性**：便于添加新功能
- **可测试性**：各层独立，便于单元测试

### 9.3 架构改进

| 方面 | 重构前 | 重构后 |
|------|--------|--------|
| 文件结构 | 单文件 1500+ 行 | 模块化多文件 |
| 路由组织 | 全部在主文件 | 蓝图分模块管理 |
| 业务逻辑 | 混合在路由中 | 独立服务层 |
| 配置管理 | 硬编码 | 配置类 + 环境变量 |
| 启动方式 | `python app.py` | `python -m app` |

### 9.4 迁移指南

如果您之前使用旧版本，请参考以下步骤迁移：

1. **备份数据**
   ```bash
   cp data/scms.db data/scms_backup.db
   ```

2. **更新启动命令**
   ```bash
   # 旧方式
   python app.py
   
   # 新方式
   python -m app
   ```

3. **配置文件迁移**
   - 原 `app.py` 中的配置已移至 `app/config/settings.py`
   - 敏感信息建议移至 `.env` 文件

4. **自定义代码迁移**
   - 如有自定义修改，请对比 `app_legacy_backup.py` 进行迁移

### 9.5 向后兼容

- 数据库格式保持不变，无需迁移
- 模板和静态资源路径保持不变
- API 接口保持不变
- 旧版本备份在 `app_legacy_backup.py` 可供参考
